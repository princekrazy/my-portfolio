# NeuroRun: Gesture Runner

An AI-powered endless runner where your webcam becomes the controller. Instead of using a keyboard, players control the game through real-time hand gestures detected with computer vision.

# Overview

NeuroRun is an experimental game that combines **computer vision**, **machine learning**, and **game development** to create a touchless gaming experience.

Using a webcam, the application detects hand gestures in real time and maps them directly to player actions inside the game.

The result is a complete AI interaction loop where physical movement controls gameplay without requiring a keyboard or controller.

---

# The Idea

Traditional games rely on physical controllers.

NeuroRun explores an alternative interaction model by allowing players to control an endless runner using only hand gestures.

The webcam continuously tracks the player's hand and translates recognized gestures into game actions with minimal latency.

---

# Features

## Real-Time Hand Tracking

The application captures live webcam footage and detects hand landmarks using **MediaPipe**.

Features include:

- Continuous hand detection
- 21 landmark tracking
- Low-latency processing
- Live gesture recognition

---

## AI Gesture Recognition

Recognized gestures are mapped directly to gameplay.

| Gesture   | Action |
| --------- | ------ |
| Open Hand | Jump   |
| Fist      | Duck   |

The gesture detection system includes smoothing techniques to improve reliability and reduce accidental inputs.

---

## Endless Runner Gameplay

The game includes:

- Physics-based jumping
- Ducking mechanics
- Procedural obstacle generation
- Increasing difficulty
- Instant restart after collision

---

## Score Tracking

Players can track:

- Current score
- High score

Scores update continuously during gameplay.

# ️ System Architecture

The gameplay is driven by a real-time processing pipeline:

```text
Webcam
    ↓
OpenCV Frame Capture
    ↓
MediaPipe Hand Detection
    ↓
Gesture Classification
    ↓
Pygame Game Loop
    ↓
Player Movement
```

Each frame passes through this pipeline, enabling smooth interaction between computer vision and gameplay.

---

# How It Works

1. Webcam captures live video.
2. OpenCV processes incoming frames.
3. MediaPipe detects 21 hand landmarks.
4. Finger positions are analyzed.
5. Gestures are classified.
6. The Pygame loop updates the player's movement instantly.

This creates a continuous AI-powered interaction loop:

> **Camera → Computer Vision → Gesture Recognition → Game Action**

# Challenges Solved

Building a gesture-controlled game introduced several technical challenges:

### Gesture Noise Reduction

Implemented buffering and smoothing techniques to reduce false gesture detections.

### Real-Time Processing

Balanced AI inference with rendering speed to maintain responsive gameplay.

### Synchronizing Systems

Coordinated webcam processing, gesture recognition, and the Pygame loop to minimize latency.

### Performance Optimization

Optimized frame processing to keep gameplay fluid while continuously analyzing webcam input.
